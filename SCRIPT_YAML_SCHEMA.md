# 剧本 YAML Schema 文档

**文档目的**：定义输出的剧本结构化数据格式，确保生成结果可读、可编辑、可被后续工具解析，并最大程度兼容影视工业习惯。 

## 1. Schema 设计核心理念

- **可读性第一**：无需任何解析工具，就能在文本编辑器中直接理解、修改剧本内容。
- **结构化与语义化**：将剧本元素（场次、对白、动作、角色）赋予明确标签，便于程序处理、统计、格式转换。
- **无损信息承载**：在满足结构化的前提下，尽量保留剧本创作所需的丰富信息，如注释、转场、括号情绪指示等。
- **可扩展性**：通过自定义字段和注释，适应不同作者、不同剧种（电影/电视剧/短剧/舞台剧）的额外需求。后续可增加更多属性

## 2. 剧本 YAML Schema 设计

### 2.1 顶层结构

```yaml
script:
  metadata:     # 剧本元信息
  characters:   # 角色清单
  scenes:       # 场景列表
```

### 2.2 完整字段说明

- **`metadata`**：剧本的基本数据，记录作品的身份信息。

  - `title`：剧本标题
  - `author`：原小说作者
  - `novel_title`：原小说名称
  - `format`：目标剧本形式（电影/电视剧/舞台剧）
  - `genre`：类型（如玄幻、武侠、言情、历史、悬疑、科幻等）
  - `notes`：全局备注，如整体改编思路、年代背景

- **`characters`**：角色列表，每个角色是一个对象。
  
  - `id`：角色的唯一标识符（用于在其他地方引用，如对话中）
  - `name`：角色名称
  - `aliases`：小说中出现的别名 
  - `role`：角色在故事中的定位（如主角、反派、配角等）
  - `description`：角色的年龄、外貌、职业、背景、性格等详细描述
  
- **`scenes`**：场景列表，每个场景记录剧本的一个完整单元。

  - `id`：场景唯一标识符
  - `chapter`：所属章节序号
  - `title`：场景标题（可选，便于识别）
  - `location`：地点
  - `time`：时间/时间段/具体时间点
  - `mood`：环境氛围或情绪基调
  - `summary`：场景内容概要
  - `beats`：场景内的叙事节拍（最小动作/对话单元）数组。每个节拍包含：
    - `id`：节拍唯一标识符。
    - `type`：节拍类型，可选值：
      - `action`：纯动作描述（无对话）。
      - `dialogue`：包含对话的节拍。
      - `transition`：转场提示。
      - `note`：写作备注或说明（或保留需要人工判断的内容）。
    - `description`：节拍的总体描述。
    - `characters_present`：参与该节拍的角色 ID 列表。
    - `dialogue`（当 `type` 为 `dialogue` 时使用）：对话列表，每句对话包含：
      - `speaker`：说话者角色 ID。
      - `line`：台词原文。
      - `emotion`：说这句台词时的情绪状态（如“愤怒”、“哽咽”）。
      - `note`：关于这句台词的额外说明（如动作指示、语调变化）。
    - `action`（当 `type` 为 `action` 时使用）：动作描述文本。
    - `note`（当 `type` 为 `note` 或补充说明时使用）：自由文本备注。
  - `transitions`：从本场景到下一个场景的转场方式数组（支持多个连续转场）。每个转场包含：
    - `type`：转场类型（`cut`/`fade`/`dissolve`/`fixed`）。
      - `cut`：直接切换。
      - `fade`：淡入淡出。
      - `dissolve`：叠化。
      - `fixed`：固定画面（无切换）。
    - `note`：关于转场的额外说明（如持续时间、特殊效果）

## 3. 设计原因说明

1. **YAML 的选择**：  
   相比 JSON，YAML 没有繁复的括号和引号，更接近自然文本，可直接用编辑器修改，而且支持注释，非常适合半自动生成的“初稿”。同时，YAML 具备严格的层级结构，易于被各种编程语言解析。

2. **层级分明，模拟剧本分页结构**：  
   `metadata → characters → scenes → beats ` 的树状结构，完美对应剧本 -> 场景 -> 节拍的逻辑，渲染成标准打印格式时几乎可以直接遍历。

3. **内容序列的类型枚举**：  
   仅用 `action`、`dialogue`、`transition` 三种内容类型，既覆盖了 95% 以上的剧本元素，又避免过度复杂。通过 `note` 类型保留了人机协作的入口，坦诚面对 AI 的不足。

4. **强调角色统一与引用**：  
   使用 ID 引用而非直接写名字，彻底解决了别名问题，使得多场景、全篇范围的角色统计、出场检查变得容易。同时通过 YAML 注释提供人类友好的名字显示。

该 YAML Schema 服务于“自动从小说生成剧本初稿”。它兼容 AI 生成与人工编辑，是整个 AI 辅助剧本创作工具的核心数据格式。
